# Retrospective: Phase 13 — Complex RPG Game UI Pages

**Date**: 2026-04-26
**Scope**: 4个新回合制RPG手游UI页面（dungeon/guild/gacha/battle），验证DSL系统在复杂场景下的表现

## 效果

| 页面 | 相似度(v1古风) | 相似度(v2赛博) | 变化 | 核心验证点 |
|------|---------------|---------------|------|-----------|
| dungeon | 94.4% | 95.3% | +0.9% | Grid布局、重复卡片、条件class |
| guild | 94.0% | 93.8% | -0.2% | 密集列表、聊天气泡对齐、Tab切换 |
| battle | 93.7% | 95.2% | +1.5% | 复合布局、HP/MP条、嵌套flex |
| gacha | 91.0% | 95.8% | +4.8% | 渐变叠加、稀有度边框 |

**风格切换效果**：赛博风v2平均95.0%，比古风v1的93.3%提升1.7%。最大提升来自gacha（+4.8%），因为减少了radial-gradient和position:absolute的使用。

## 原因分析

### 成功的部分

1. **Grid布局稳定**：dungeon的4列关卡网格和battle的3列技能栏都正确渲染。Grid是DSL最可靠的布局之一。

2. **HP/MP进度条**：battle页面的widthPercent进度条完美工作，渐变背景+百分比宽度是Phase 11已验证的特性，这里再次确认可靠。

3. **CSS class条件样式**：dungeon的cleared/current/locked三种状态通过不同CSS class区分，选择器引擎正确处理了`.stage-card.cleared`等复合选择器。

4. **密集列表布局**：guild的成员列表（6行重复结构+分隔线）和chat区域（左/右/系统三种气泡）都能正确渲染。

### 失败/不足的部分

5. **gacha radial-gradient**：CSS `radial-gradient()` 不在当前支持范围内。召唤阵区域渲染为纯色背景而非径向渐变。**根因**：`css_mapper.py`仅支持`linear-gradient`。

6. **gacha position:absolute**：召唤阵外围的发光圈使用了`position:absolute`叠加，DSL不支持absolute定位。元素按正常文档流排列。**根因**：absolute定位是设计文档中已标记的"不支持"特性。

7. **gacha border-radius:50%**：CSS `border-radius:50%` 生成圆形，但DSL的`cornerRadius`仅支持px值。召唤阵渲染为圆角矩形而非圆形。**根因**：`css_mapper.py`中`border-radius`解析只处理px，不处理百分比。

8. **battle filter:brightness()**：当前行动角色的`.active-turn`用了`filter:brightness(1.2)`高亮，DSL不支持CSS filter。高亮效果仅通过金色边框+box-shadow体现。**根因**：CSS filter不在DSL特性列表中。

### 量化影响

gacha页面最低（91.0%），差距主要来自：
- 召唤阵圆形→圆角矩形（~2%）
- radial-gradient→纯色背景（~1%）
- position:absolute overlay缺失（~0.5%）

## 方向反思

### DSL能力评估

经过11个页面的验证，DSL系统的**核心布局能力足够**：
- Flex（vertical/horizontal）✓
- Grid ✓
- 进度条（widthPercent）✓
- 渐变背景/文字 ✓
- 条件样式（CSS class选择器）✓
- Box shadow ✓

**明确的差距**：
- `border-radius:50%` — 可低成本实现（扩展css_mapper解析%）
- `radial-gradient` — 中等成本（需要新的渐变解析+渲染路径）
- `position:absolute` — 高成本（需要全新的布局模式）
- CSS filter — 高成本（Qt无原生支持）

### 建议优先级

1. **border-radius百分比支持** — 影响gacha -3%，实现成本低（改1个正则+后端映射）
2. **接受91-94%为新页面的合格线** — 与Phase 12结论一致，Qt有天花板
3. **优先引擎后端** — 这些差距（radial-gradient、position:absolute）在引擎UI库中可能不存在
4. **暂不实现radial-gradient** — 用途局限（主要是圆形渐变），投入产出比低

### 结论

Phase 13成功验证了DSL在复杂RPG游戏UI场景下的可用性。4个页面从简到难，覆盖了真实手游的核心UI模式。最低分91.0%（gacha）主要受限于2个已知的DSL不支持的CSS特性。**DSL系统的布局和样式能力通过了验证，可以进入引擎后端实现阶段。**
