# Complex RPG Game UI Pages — Design Spec

**Date**: 2026-04-26
**Goal**: 设计4个复杂回合制手游UI页面，验证DSL系统在真实游戏场景下的表现力

## 统一视觉风格

古风深色主题，参考幻唐志手游UI风格：

| 色彩角色 | 色值 | 用途 |
|----------|------|------|
| 主背景 | `#1a0f0a` | 页面底色 |
| 卡片背景 | `#2a1a10` | 面板、卡片底色 |
| 金色强调 | `#c9a034` / `#dbb850` | 标题、边框、重要按钮 |
| 暗红强调 | `#8b2020` / `#a03030` | HP条、危险操作、稀有边框 |
| 正文文字 | `#e8d5b5` | 主要文字 |
| 次要文字 | `#a89070` | 描述、标签 |
| 按钮高亮 | `#3a6a30` | 确认操作 |
| 按钮普通 | `#4a3a28` | 普通操作 |

圆角统一 `8px`，间距统一 `8px`/`12px`，按钮高度 `40px`。

## 页面1：副本选择 (dungeon.html)

**复杂度**：★★★☆☆ — 网格布局 + 星级评分 + 进度展示

### 布局结构

```
┌──────────────────────────────┐
│  ← 副本选择    章节：第三章     │  ← 顶栏
├──────────────────────────────┤
│  ┌─────┬─────┬─────┬─────┐  │
│  │ 1-1 │ 1-2 │ 1-3 │ 1-4 │  │  ← 章节关卡网格
│  │ ★★★ │ ★★☆ │ ★☆☆ │ 🔒 │  │     星级 + 锁定状态
│  ├─────┼─────┼─────┼─────┤  │
│  │ 1-5 │ 1-6 │ 1-7 │ 1-8 │  │
│  │ ★★★ │ ★★☆ │ 🔒  │ 🔒 │  │
│  └─────┴─────┴─────┴─────┘  │
├──────────────────────────────┤
│  ┌──────────────────────┐    │
│  │ 章节1 │ 章节2 │ 章节3 │    │  ← 章节标签页
│  └──────────────────────┘    │
├──────────────────────────────┤
│  [扫荡10次]  [进入战斗]       │  ← 底部操作栏
└──────────────────────────────┘
```

### 关卡卡片设计

每个关卡卡片：
- 背景：`#2a1a10`，金色边框（已通关）/ 灰色边框（未解锁）
- 关卡编号：金色 `#c9a034`
- 星级：已获得星 `#c9a034`，未获得星 `#4a3a28`
- 锁定状态：半透明遮罩 + 🔒 图标
- 尺寸：~80x90px

### 验证重点

- **Grid布局**：`display:grid; grid-template-columns: repeat(4, 1fr)` → DSL grid layout
- **重复卡片模式**：同一结构重复多次，测试CSS选择器级联
- **条件样式**：已通关/未解锁两种状态用不同CSS class区分

## 页面2：公会 (guild.html)

**复杂度**：★★★☆☆ — Tab切换 + 密集列表 + 聊天气泡

### 布局结构

```
┌──────────────────────────────┐
│  公会名称     [退出公会]      │  ← 顶栏
├──────────────────────────────┤
│  [成员] [动态] [设置]         │  ← Tab栏
├──────────────────────────────┤
│  ┌──────────────────────┐    │
│  │ 公会公告栏            │    │  ← 公告区
│  │ 欢迎加入...           │    │
│  └──────────────────────┘    │
├──────────────────────────────┤
│  ┌──────────────────────┐    │
│  │ 👑 会长  Lv.60  在线 │    │  ← 成员列表
│  ├──────────────────────┤    │
│  │ ⚔ 副会长 Lv.58  在线 │    │     每行：职位图标 + 名字 + 等级 + 状态
│  ├──────────────────────┤    │
│  │ ☆ 成员  Lv.45  离线 │    │
│  ├──────────────────────┤    │
│  │ ☆ 成员  Lv.42  在线 │    │
│  ├──────────────────────┤    │
│  │ ☆ 成员  Lv.38  离线 │    │
│  └──────────────────────┘    │
├──────────────────────────────┤
│  ┌──────────────────────┐    │
│  │ [玩家A]: 打副本吗？   │    │  ← 聊天区
│  │       [玩家B]: 来！  │    │     左/右气泡
│  │ [系统]: 公会战即将开始│    │
│  └──────────────────────┘    │
│  [输入消息...        ] [发送] │  ← 输入栏
└──────────────────────────────┘
```

### 成员行设计

每行高度 48px：
- 左侧：职位图标（👑⚔☆）+ 角色名
- 右侧：等级 + 在线状态（绿色圆点/灰色）
- 分隔线：`border-bottom: 1px solid #3a2a18`

### 聊天气泡

- 左侧气泡（他人）：`#3a2a18` 背景，左对齐，左圆角
- 右侧气泡（自己）：`#2a4a28` 背景，右对齐，右圆角
- 系统消息：居中，`#a89070` 小字

### 验证重点

- **密集列表**：重复行结构 + 分隔线
- **聊天气泡**：不同对齐方向的flex布局
- **Tab栏**：水平按钮组 + 选中态高亮
- **flex布局混合**：同一页面多种flex排列（row/column）

## 页面3：抽卡/召唤 (gacha.html)

**复杂度**：★★★★☆ — 渐变叠加 + 稀有度边框 + 动画暗示

### 布局结构

```
┌──────────────────────────────┐
│  ← 召唤      [抽卡记录]      │  ← 顶栏
├──────────────────────────────┤
│                              │
│     ┌──────────────┐        │
│     │              │        │
│     │   法阵图案    │        │  ← 中央召唤区（渐变圆形）
│     │  (radial     │        │     radial-gradient 暗示
│     │  gradient)   │        │
│     │              │        │
│     └──────────────┘        │
│                              │
│  [限定池] [常驻池] [友情池]   │  ← 卡池Tab
│                              │
│  限定池：SSR 3%  SR 15%      │  ← 概率显示
├──────────────────────────────┤
│  [单抽×1]    [十连抽×10]     │  ← 操作按钮
│   200钻石     1800钻石       │
├──────────────────────────────┤
│  ┌───┬───┬───┬───┬───┐      │
│  │ ★ │ ★ │★★│★★│★★★│      │  ← 结果展示区（十连结果）
│  └───┴───┴───┴───┴───┘      │     按稀有度显示不同边框颜色
└──────────────────────────────┘
```

### 召唤阵设计

中央区域：
- 圆形容器：`border-radius: 50%`
- 背景：`radial-gradient`（如支持）或多层渐变叠加
- 发光效果：`box-shadow` 多层阴影

### 稀有度边框

| 稀有度 | 边框色 | 效果 |
|--------|--------|------|
| N (★) | `#8a7a60` | 普通 |
| R (★★) | `#4a8aff` | 蓝色边框 |
| SR (★★★) | `#c850ff` | 紫色边框 + 渐变文字 |
| SSR (★★★★) | `#ffa500` | 金色渐变边框 + 发光 |

### 验证重点

- **渐变叠加**：多层背景渐变（测试radial-gradient支持或降级方案）
- **稀有度系统**：不同CSS class对应不同边框颜色方案
- **渐变文字**：SSR稀有度名称使用 `background-clip:text` 渐变
- **圆形布局**：`border-radius:50%` 大圆元素
- **Grid展示**：十连结果5x2网格

## 页面4：战斗界面 (battle.html)

**复杂度**：★★★★★ — 最复杂页面，综合验证所有DSL能力

### 布局结构

```
┌──────────────────────────────┐
│  回合 3/10    [自动] [加速]   │  ← 顶栏：回合数 + 控制按钮
├──────────────────────────────┤
│                              │
│     ┌───┐  ┌───┐  ┌───┐    │
│     │敌1│  │敌2│  │敌3│    │  ← 敌方站位（3个）
│     │HP │  │HP │  │HP │    │     HP条在角色下方
│     └───┘  └───┘  └───┘    │
│                              │
│  ────────速度条──────────    │  ← 速度条/行动顺序
│  敌1 → 我2 → 敌3 → 我1      │
│                              │
│     ┌───┐  ┌───┐  ┌───┐    │
│     │我1│  │我2│  │我3│    │  ← 我方站位（3个）
│     │HP │  │HP │  │HP │    │
│     │MP │  │MP │  │MP │    │     HP + MP 双条
│     └───┘  └───┘  └───┘    │
├──────────────────────────────┤
│  ┌──────┬──────┬──────┐     │
│  │ 普攻 │ 技能1 │ 技能2 │     │  ← 技能栏
│  ├──────┼──────┼──────┤     │
│  │ 技能3 │ 技能4 │ 防御 │     │
│  └──────┴──────┴──────┘     │
│  ┌──────────────────────┐    │
│  │ 战斗日志...           │    │  ← 底部战斗日志
│  └──────────────────────┘    │
└──────────────────────────────┘
```

### 角色卡片设计

每个角色卡片 ~80x100px：
- 背景：`#2a1a10`
- 角色占位：`#3a2a18` 圆形区域（模拟头像）
- HP条：`#8b2020` → `#cc3030`（渐变红）+ 数值文字
- MP条：`#2040a0` → `#3060cc`（渐变蓝）+ 数值文字
- 边框：当前行动角色金色发光边框（`box-shadow`）

### 速度条

- 横向flex布局
- 角色头像小圆（32x32）+ 箭头指示行动顺序
- 当前行动者高亮

### 技能栏

- 2x3 Grid布局
- 每个技能按钮：
  - 普通：`#4a3a28` 背景
  - 可用：金色边框
  - 冷却中：半透明遮罩 + 回合数
  - MP不足：灰色

### 验证重点

- **复合布局**：页面内同时使用 vertical/horizontal/grid 三种布局
- **HP/MP进度条**：渐变背景 + 百分比宽度（`widthPercent`）
- **Grid技能栏**：`grid-template-columns: repeat(3, 1fr)`
- **状态指示**：通过CSS class控制不同视觉状态
- **box-shadow**：当前行动角色的发光边框
- **嵌套flex**：角色卡片内垂直排列（头像 + HP + MP）
- **战斗日志**：多行文本 + 滚动区域暗示

## 实施顺序

按复杂度递增实施，每页完成后转DSL + 视觉测试：

1. **dungeon.html** — 网格布局基础验证
2. **guild.html** — 列表 + 多flex布局验证
3. **gacha.html** — 渐变 + 稀有度系统验证
4. **battle.html** — 全特性综合验证

## 预期DSL能力扩展

这4个页面可能暴露当前DSL不足之处，需要评估是否扩展：

| CSS特性 | 当前状态 | 预计需求 |
|---------|---------|---------|
| `display:grid` | 已支持 | dungeon关卡网格、battle技能栏 |
| `border-radius:50%` | 仅支持px | gacha召唤阵需要 |
| `radial-gradient` | 不支持 | gacha召唤阵（可降级） |
| `position:absolute` | 不支持 | battle角色定位（可降级为grid） |
| `overflow:hidden` | 不支持 | battle日志区（可降级） |
| `text-overflow:ellipsis` | 不支持 | guild成员名（可降级） |
| `:hover` 伪类 | 不支持 | 按钮悬停效果（可降级） |

所有页面设计为「降级友好」——不依赖 `position:absolute` 或 `overflow` 也能完成核心布局。
